# PyCon 인터랙티브 스폰서 게임 기획서 v1

## 1) 프로젝트 한 줄 요약
**관객 채팅 참여 + 후원사 스킬**로 매 라운드 흐름이 바뀌는, 행사형 실시간 미니게임.

---

## 2) 목표
- PyCon 참가자에게 즉각적인 재미와 참여감 제공
- 후원사 노출을 배너가 아닌 **게임 메커닉**으로 체감시킴
- 2~4분 라운드 기반으로 부스/세션 쉬는 시간에 반복 운영 가능하게 구성

---

## 3) 게임 컨셉 (Working Title)
**Bug Miner: Sponsor Edition**

플레이어(자동/수동)가 아래로 내려가며 블록을 파괴하고 점수를 쌓는다.
관객은 채팅 명령으로 버프/이벤트를 발생시키고,
후원사 스킬은 특정 조건 또는 쿨다운마다 발동된다.

---

## 4) 코어 게임 루프
1. 라운드 시작 (카운트다운 3초)
2. 채굴/회피/폭발 이벤트 진행
3. 관객 채팅 큐 처리(명령 반영)
4. 후원사 스킬 발동(조건/쿨다운)
5. 라운드 종료(2~4분 또는 HP 0)
6. 결과 화면(MVP 참여자, 점수, 스폰서 하이라이트)

---

## 5) 관객 참여 규칙
### 기본 명령어 (초안)
- `tnt`: TNT 1개 생성
- `boost`: 이동/공격 속도 상승(짧게)
- `slow`: 게임 속도 저하(짧게)
- `big`: 곡괭이/충돌 범위 확대
- `shield`: 보호막 1회

### 운영 규칙
- 유저별 중복 입력 쿨다운: 10~20초
- 전역 큐 처리 주기: 2~5초
- 한 번에 반영되는 이벤트 개수 제한(폭주 방지)

---

## 6) 후원사 스킬 프레임워크
후원사별로 1개의 시그니처 스킬 + 짧은 시각 효과 + 설명 문구 노출.

예시)
- **Cloud Sponsor / AutoScale**: 8초간 자원 드롭 증가
- **Security Sponsor / ShieldWall**: 함정 1회 무효 + 경고 이펙트
- **AI Sponsor / SmartPath**: 5초간 최적 경로 하이라이트

원칙:
- 스킬 지속시간 짧게(3~8초)
- 사기성 방지(쿨다운, 상쇄 리스크)
- 브랜드 연상 포인트는 1개만 강하게

---

## 7) 점수/보상 시스템
- 기본 점수: 파괴 블록 가치 합
- 콤보 점수: 연속 파괴 시 배수 증가
- 참여 보너스: 채팅 명령 채택된 유저 기여도 집계
- 라운드 종료 화면:
  - 총점
  - Top 참여자 3명
  - 후원사 스킬 기여도(발동 횟수/효과)

---

## 8) UI/연출
- 송출 기본 해상도: 9:16 (쇼츠/릴스 친화)
- 선택 지원: 16:9 오버레이 모드
- 항상 표시:
  - 남은 시간 / HP / 점수
  - 최근 채팅 명령 로그
  - 현재 활성 스킬
- 이벤트 발생 시 1~2초 스폰서 카드 노출

---

## 9) 기술 스택 제안
- Python + Pygame + Pymunk
- 채팅 연동: YouTube Live Chat API (후속으로 Twitch/Discord 확장 가능)
- 구성 파일 기반 밸런싱: `config.json`
- 런타임 장애 대응: 자동 재시작 스크립트(run.sh/run.ps1)

---

## 10) MVP 범위 (2주)
### 포함
- 물리 기반 플레이 루프
- 채팅 명령 4~5개
- 후원사 스킬 3개
- 결과 화면 + 기본 로그 저장
- OBS 송출용 장면 구성 가이드

### 제외(후속)
- 계정 연동 랭킹
- 웹 대시보드 실시간 운영툴
- 다국어 현장 운영 UI

---

## 11) 개발 마일스톤
### M1 (D1~D3)
- 프로젝트 스캐폴딩
- 코어 물리/카메라/점수 루프 구현

### M2 (D4~D6)
- 채팅 큐 + 명령 처리 + 쿨다운
- 기본 UI/HUD

### M3 (D7~D9)
- 후원사 스킬 시스템(플러그인형)
- 밸런스 파라미터 분리

### M4 (D10~D12)
- 결과 화면 / 로그 / 안정화
- 크래시 복구 및 운영 스크립트

### M5 (D13~D14)
- 리허설 모드(자동 입력)
- 행사 시나리오 테스트 + 튜닝

---

## 12) 리스크와 대응
- API 할당량/지연: 폴링 간격 조절 + fallback(오프라인 이벤트)
- 채팅 폭주: 큐 제한/샘플링/쿨다운
- 밸런스 붕괴: 서버측 상한 및 긴급 밸런스 핫키
- 현장 장애: 자동 재시작 + 데모 모드 준비

---

## 13) 데모 운영 시나리오 (현장)
1. 사회자: “채팅에 `tnt` 입력해보세요!”
2. 관객 명령 반영 + 폭발 연출
3. 후원사 스킬 발동 멘트 + 카드 노출
4. 3분 후 결과 발표 + 다음 라운드 안내
