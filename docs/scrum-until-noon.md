# 12시까지 10분 스크럼 루프 실행 계획

## 0) 목적/원칙
- 목적: **12:00 데모 가능 상태(MVP flow end-to-end)** 달성
- 루프 길이: 10분 (계획 2분 → 실행 6분 → 확인/기록 2분)
- 우선순위: **통합 가능한 얇은 수직 슬라이스**를 먼저 완성
- 산출물 기준: 각 루프 종료 시 `증거(커밋/스크린샷/로그)` 1개 이상 남김

---

## 1) 디자이너/개발자 산출물 통합 순서 (고정)
1. **UX 흐름 고정** (디자이너)
   - 핵심 화면 3개(입력/처리중/결과) 와이어 또는 하이파이 확정
2. **디자인 토큰 확정** (디자이너)
   - 색/타이포/간격/상태(성공·오류·로딩)
3. **컴포넌트 뼈대 구현** (개발자)
   - 버튼/입력/카드/로더 등 재사용 컴포넌트
4. **핵심 플로우 연결** (개발자)
   - 입력 → 처리 → 결과의 데이터 전달 연결
5. **콘텐츠/문구 반영** (디자이너 주도, 개발자 적용)
   - 빈 상태/오류 메시지/CTA 문구
6. **품질 마감** (공동)
   - 반응형/기본 접근성/치명 버그/데모 시나리오 리허설

> 원칙: 시각 완성도보다 **동작 일관성** 우선. 디자인 디테일은 후반 루프에서 증분 반영.

---

## 2) 10분 사이클 운영 규칙
- 매 사이클 시작 질문(30초):
  1) 이번 10분에서 사용자 가치가 늘어나는가?
  2) 끝났음을 객관적으로 어떻게 증명할 것인가?
- WIP 제한: 각 역할 **동시 1작업만**
- 블로커 2분 초과 시: 즉시 우회안 선택(스코프 축소/목업 데이터 사용)
- 사이클 종료 시 기록:
  - 완료/미완료
  - 다음 사이클 단일 목표
  - 데모 리스크 1개

---

## 3) 12시까지 타임박스 계획 (10분 루프)

> 아래는 `T-` 기준 템플릿. 현재 시각에 맞춰 가장 가까운 루프부터 시작.

### Phase A — 기반 고정 (T-120 ~ T-90)
**Cycle A1 (10m)**
- 목표: 데모 핵심 사용자 시나리오 1개 확정
- 완료조건: "누가-무엇을-어떤 결과로" 1문장 + 화면 3개 정의

**Cycle A2 (10m)**
- 목표: 화면 IA/전환 정의
- 완료조건: 화면 전환 다이어그램(입력→처리→결과) 1개

**Cycle A3 (10m)**
- 목표: 디자인 토큰 1차 확정
- 완료조건: 색/타입/스페이싱/상태 토큰 표 작성

### Phase B — 구현/연결 (T-90 ~ T-40)
**Cycle B1 (10m)**
- 목표: 공통 레이아웃/컴포넌트 뼈대 생성
- 완료조건: 버튼/입력/카드/로더 렌더 확인

**Cycle B2 (10m)**
- 목표: 입력 화면 동작
- 완료조건: 폼 입력/검증/제출 이벤트 동작

**Cycle B3 (10m)**
- 목표: 처리중 상태 연결
- 완료조건: 로딩 상태 진입/이탈 + 실패 핸들링 분기 1개

**Cycle B4 (10m)**
- 목표: 결과 화면 연결
- 완료조건: 목업 또는 실제 데이터 1건 표시

**Cycle B5 (10m)**
- 목표: 디자인 문구/피드백 반영
- 완료조건: 빈 상태/오류/성공 문구 적용 + CTA 1개 확정

### Phase C — 안정화/데모 준비 (T-40 ~ T-0)
**Cycle C1 (10m)**
- 목표: 통합 테스트 1차
- 완료조건: 핵심 시나리오 end-to-end 1회 무중단 성공

**Cycle C2 (10m)**
- 목표: 버그 컷(치명도 기준)
- 완료조건: P0=0, P1 최대 2개 이하

**Cycle C3 (10m)**
- 목표: 데모 스크립트 작성/리허설
- 완료조건: 3분 데모 대본 + 클릭 경로 + 예상 질문 3개

**Cycle C4 (10m, 버퍼)**
- 목표: 예비시간(리스크 대응)
- 완료조건: 남은 결함 수정 또는 시각 polish 1개

---

## 4) 역할별 즉시 할 일 (동시 진행)
### 디자이너
- A1~A3: 핵심 흐름/토큰/카피 가이드 확정
- B5: 마이크로카피 및 상태별 메시지 최종화
- C단계: 데모 화면 가독성(대비/간격/강조) 점검

### 개발자
- B1~B4: 컴포넌트/상태/데이터 연결 우선
- C1~C2: E2E 기준 안정화 및 치명 버그 제거
- C3~C4: 데모 클릭 경로 단축(불필요 단계 제거)

---

## 5) 데모 기준(Definition of Demo Ready)
아래 6개 중 6개 충족 시 데모 진행:
1. 핵심 사용자 시나리오 1개를 **3분 내 시연 가능**
2. 입력→처리→결과 플로우가 **중단 없이 1회 이상 성공**
3. 로딩/오류/빈 상태 중 최소 2개가 의도대로 동작
4. 치명 버그(P0) 0건
5. 데모 담당자 외 팀원 1명이 동일 경로 재현 성공
6. 데모 스크립트/백업 시나리오(실패 시 대체 화면) 준비 완료

---

## 6) 리스크 및 우회 전략
- API/백엔드 지연: 목업 응답으로 데모 경로 고정
- 디자인 확정 지연: 토큰 최소셋(Primary/Gray/Error + 2 텍스트 크기)으로 축소
- 통합 충돌: 브랜치 전략 단순화(`main` 직전 통합 브랜치 1개)
- 시간 초과: 신규 기능 추가 중단, 데모 경로 안정화에 전량 투입
